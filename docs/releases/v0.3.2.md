## Hard rules

`maddu doctor` enforces the 8 invariants on every install + upgrade:

1. **Files-only state** — no `*.db`/`*.sqlite` under `.maddu/`
2. **No DB packages** — no `sqlite3` / `better-sqlite3` family in `package.json`
3. **No hosted backends** — provider calls happen in subprocesses
4. **No broad new dependencies** — Node stdlib only
5. **No provider SDKs in app code** — scan `maddu/` for `anthropic`/`openai`/`@google/generative-ai`
6. **No token leaks under `.maddu/`** — regex scan for AKIA/sk-/ghp_/AIza/xox patterns
7. **Three-layer brand boundary** — documented, hand-enforced
8. **Lane ownership** — no duplicate active claims
9. **State containment** *(Slice 17 addition)* — no Máddu state dirs at repo root

## Security invariants verified end-to-end

- **No raw API keys** ever appear in spine, projections, or anywhere under `.maddu/`.
- **Auth library boundary** — `maskRecord()` drops the `value` field before any HTTP response.
- **Import gateway** rejects whole payloads on first secret-shape hit. Ledger stores **paths + pattern names only**, never values.

## What's not in v0.3.2

- **D3** (office-artifact preview) — deferred; adds rendering dependencies. Revisit when a concrete use case demands it.
- **Active key consumption by spawned workers** — `auth.activeValue()` is wired to inject keys at spawn time, but no Máddu-spawned worker currently consumes a provider key. The plumbing is ready; a first-party runtime adapter will activate it.
