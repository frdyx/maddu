## Notes from the walk

- **Auth store is device-global, not repo-bound.** Keys added in one repo are
  visible to `maddu auth list` from any other Máddu repo on the same machine
  — by design (per Phase C5: tokens are device credentials). The smoke
  re-added two keys; the listing shows the cumulative provider state across
  prior runs.
- **Schedule tick at 18:23 didn't match the `every monday at 9am` cron** —
  expected; the cron was correctly parsed but the wallclock didn't fall on
  a Monday-9am minute. To trigger fire-during-smoke, use `every minute` or
  pass `--at <iso>` to `schedule tick`.
- **Runtime spawn forked a real subprocess** (pid 39448 in this run). Log
  file landed at `.maddu/state/worker-logs/<workerId>.log` — confirming
  Phase B5's heartbeat path and Phase C1's spawn pipeline are wired.
- **Import rejection caught both regex-pattern and sensitive-field-name
  detection paths.** The dirty payload had `config.api_key` (caught by
  both `sensitive-field` and `anthropic-key` patterns) plus `config.fine`
  matching `github-pat`. The rejection ledger recorded the JSON paths and
  pattern names — never the values.
